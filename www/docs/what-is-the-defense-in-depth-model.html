Title: What is the Defense In Depth Model?

Defense is like an onion, which have layers you see. In a secure environment, there should be layers of defense to protect an asset. For example, there should be layers of defense in place to protect data on a server. The data should have file permissions defined which only allow users access who have a need-to-know about the data. The data should be encrypted. The server should be stored in a secure room. The secure room should have a security camera and locked door. As you understand I'm sure, there should be a list of layers which protect an asset. If there were a single point of failure, then we would be poor custodians of this data. The main objective is to not have a single point of failure.
